# Orbit Agents 宣傳文

我有用 n8n 長期做一些排程，整理 Sentry Log 或 Google Analytics 的報告並寄送給自己，但 n8n 放在雲端需要一筆費用，節點的串接門檻也高 (我也只會 fork 別人的來改 XD)，需要什麼資料一開始就要定義清楚，相對來說，與 AI 的溝通還是簡單一些

後來用 Sentry MCP 和 GA MCP，它可以很好的整合我要的資料並進行建議；但因為 CLI 沒辦法做排程，所以我做了一個小工具：Orbit Agents，功能是搭配 cron 表達式呼叫本地端的 CLI 進行排程任務，100% 開源

我目前整合了 Claude Cli 和 Gemini Cli，手上沒 Codex 所以暫時不支援，歡迎大家發 PR XXD

他的使用方式很簡單，**只需要三個步驟**：

1. **選擇 AI 模型**：Claude (Sonnet/Opus/Haiku) 或 Gemini (3/2.5/2)
2. **寫下你的需求**：用自然語言告訴 AI 你要做什麼，例如「幫我整理這週的 Sentry 錯誤，找出最嚴重的三個問題並分析原因」
3. **設定排程時間**：選擇每天、每週、每月，或自訂 cron 表達式

就這樣！不需要學習複雜的節點串接，不需要一開始就定義好所有資料結構，AI 會根據你的 prompt 和 MCP 工具自動整合資料、產生報告，並在執行完成後透過 Email 寄送給你。

**為什麼選擇 Orbit Agents？**

- 🆓 **完全免費**：本地運行，不需要支付任何雲端服務費用
- 🤖 **AI 驅動**：用自然語言溝通，比視覺化節點更直覺
- 🔌 **MCP 整合**：支援 Sentry、Google Analytics 等 MCP 工具，輕鬆整合各種資料來源
- 📊 **完整記錄**：每次執行都有詳細的日誌，成功失敗一目了然
- 🖥️ **桌面應用**：macOS、Windows、Linux 全平台支援，資料完全掌控在自己手中
- 🔓 **100% 開源**：MIT License，歡迎貢獻、改進、fork

如果你也厭倦了雲端服務的費用和複雜的節點配置，想要一個更簡單、更靈活的排程工具，不妨試試 Orbit Agents！
